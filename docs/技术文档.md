# VibeLive 技术文档

> 一个 AI 自制氛围生成器：用户用一句话描述场景 → AI 生成曲子参数、音乐 pattern 与动态氛围页面 → 浏览器实时合成无限循环音乐。

本文档基于产品规划（见 [`docs/想法.md`](./想法.md)）与当前已落地的前端代码编写，描述**现状架构**与**演进路线**，供后续开发参与者快速理解项目全貌。

---

## 1. 项目概览

### 1.1 产品定位

VibeLive 的产品核心是「自制」：用户不需要懂 DAW、合成器或 prompt 工程，只需要简要描述一个场景、情绪、时间、质感或画面，AI 就生成一首可循环的曲子和对应的动态氛围页面。

它不是传统 DAW，也不是直接生成固定 mp3 的音乐模型产品。核心路线是：

- AI 通过 OpenAI SDK 调用火山引擎方舟 OpenAI 兼容 API，把自然语言转成结构化 `Vibe`。
- `Vibe` 同时包含音乐参数、pattern DSL、调色板、内置视觉类型和可选 AI 视觉 HTML。
- 浏览器用 Tone.js 实时合成无限循环音乐，而不是播放固定音频文件。
- 页面优先展示 AI 生成的 HTML/Canvas 视觉；不安全或失败时回退到内置 Canvas 视觉。

一句话定位：

> 用一句描述生成一首曲子和一个会呼吸的氛围页面。

### 2.1 OpenAI 兼容配置

产品侧支持用户在首页 `AI 配置` 面板填写 API Key、Base URL、Model，并保存到当前设备的 `localStorage`。生成请求会把这份配置作为请求级覆盖传给本地 API 代理，服务端不写入文件，也不在返回体暴露 key。

开发/部署侧仍可通过官方 `openai` SDK 接 OpenAI 兼容 Chat Completions API。优先读取用户请求级配置，其次读取 `OPENAI_*`，旧的火山引擎 Ark `ARK_*` 变量仍然兼容；当只配置 Ark key 且未配置 base URL 时，默认使用：

```txt
https://ark.cn-beijing.volces.com/api/v3
```

本地配置见 `.env.local.example`：

```bash
OPENAI_API_KEY=your-openai-compatible-api-key
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4.1-mini
VIBELIVE_MAX_CONVERSATION_MESSAGES=8
NEXT_PUBLIC_VIBELIVE_MAX_CONVERSATION_MESSAGES=8

ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
ARK_API_KEY=your-volcengine-ark-api-key
ARK_MODEL=your-ark-endpoint-id-or-model-name
```

## 4. 核心数据模型

数据模型是整个产品的基石——一个 vibe 的完整声音与页面 = 一个可序列化对象。这是 AI 自制、分享链接、preset 导出和社区分享的共同基础。

定义于 `src/data/vibes.ts`：

```ts
interface VibeParams {
  energy: number;     // 整体能量 / 律动强度 0~1
  warmth: number;     // 音色温暖度（低=明亮数字感，高=温暖模拟感）0~1
  space: number;      // 空间 / 混响大小 0~1
  brightness: number; // 明亮度 / 高频含量 0~1
  ambience: number;   // 环境音强度（雨声、风声等）0~1
  density: number;    // 音符密度 / 繁复程度 0~1
  tempo: number;      // 速度 BPM
}

interface VibePalette {
  accent: string;   // 主强调色
  accent2: string;  // 次强调色
  base: string;     // 背景基调（近黑，带冷暖倾向）
}

interface VibePattern {
  root: string;
  scale: "major" | "minor" | "minorPentatonic" | "majorPentatonic" | "dorian" | "lydian";
  chords: string[];
  seed: number;
  layers: Record<"drums" | "bass" | "pad" | "melody" | "arp", {
    enabled: boolean;
    density: number;
    swing: number;
    octave?: number;
  }>;
  mini?: {
    drums?: Partial<Record<"kick" | "snare" | "hat" | "percussion", string>>;
    bassline?: string;
    melodyMotif?: string;
    arpPattern?: string;
  };
  strudel?: {
    version: 1;
    code: string;
    notes?: string;
  };
}

interface Vibe {
  id: string;
  name: string;       // 中文展示名
  subtitle: string;   // 英文副名
  tagline: string;    // 一句诗意的氛围描述
  glyph: string;      // 卡片符号
  visual: VisualType; // "orbs" | "rain" | "particles" | "waves" | "cosmos"
  palette: VibePalette;
  params: VibeParams;
  pattern?: VibePattern;
  visualCode?: string;
  source?: "builtin" | "ai";
}
```

### 内置氛围（6 个）

| id | 展示名 | 视觉风格 | 特征参数 |
| --- | --- | --- | --- |
| `cyber-night` | 深夜写代码 | orbs | 能量偏高，冷色，92 BPM |
| `rainy-lofi` | 雨天阅读 | rain | 温暖、环境音强，72 BPM |
| `cyber-meditation` | 赛博冥想 | waves | 极大空间，低密度，58 BPM |
| `deep-focus` | 低保真学习 | particles | 暖色稳定，80 BPM |
| `forest-walk` | 森林散步 | particles | 自然绿，环境音强，76 BPM |
| `cosmic-drift` | 宇宙漂浮 | cosmos | 空间拉满，极慢 50 BPM（首页默认底色） |

### 辅助导出

- `PARAM_META`：参数到 UI 的映射表（标签、min/max、单位），供 `PlayerView` 渲染滑块时遍历，**新增参数只需在此登记**。
- `getVibe(id)`：按 id 查找 vibe（为分享链接 / 路由预留）。

## 5. 状态管理与数据流

当前应用是**单页面、无路由切换**的，全部状态集中在 `page.tsx` 的 `Page` 组件中，使用 React `useState`：

```ts
const [active, setActive]   = useState<Vibe | null>(null);  // null = 首页；非空 = 播放器
const [preview, setPreview] = useState<Vibe>(HOME_VIBE);    // 首页选择器悬停/聚焦的预览
const [params, setParams]   = useState<VibeParams>(HOME_VIBE.params); // 当前可调参数
const [playing, setPlaying] = useState(false);
```

### 数据流向

```txt
                       ┌─────────────────────────────────────────┐
                       │              Page (状态中枢)              │
                       │  active / preview / params / playing     │
                       └───────┬───────────────────────┬─────────┘
        onPreview / onSelect   │                       │  params / playing
                               ▼                       ▼
   ┌───────────────────────────────┐     ┌──────────────────────────┐
   │   VinylSceneSelector (首页)    │     │   PlayerView (播放器)     │
   │   - 切换唱片 → onPreview        │     │   - onParamChange         │
   │   - 点播放   → onSelect         │     │   - onTogglePlay          │
   └───────────────────────────────┘     │   - onBack                │
                                          └──────────────────────────┘
                               │ displayVibe = active ?? preview
                               ▼
                     ┌──────────────────────┐
                     │   VibeBackground      │  ← 实时视觉，消费 vibe + params + playing
                     └──────────────────────┘
```

关键逻辑：

- `displayVibe = active ?? preview`：未进入播放器时，背景跟随选择器预览；进入后跟随选中的 vibe。
- `selectVibe(vibe)`：设置 `active`、复制 `vibe.params` 到可调状态、`playing = true`。
- `changeParam(key, value)`：不可变更新单个参数，立即反映到视觉与代码预览。

> **演进建议**：参数与播放状态后续会被音频引擎、分享 URL、preset 存储等多处消费。当跨组件共享需求增多时，建议迁移到 Zustand/Jotai（规划已列出），把 `params` 作为单一可信源（single source of truth）。

### 5.1 AI 自制端到端流程

```txt
用户描述
  ↓
VibePrompt
  ↓ POST /api/generate
src/app/api/generate/route.ts
  ↓ openai SDK（baseURL = 火山方舟 /api/v3）
Doubao / Ark model
  ↓ JSON schema 结构化输出
src/ai/schema.ts：Zod 校验、clamp、兜底
  ↓
Page.setActive(generatedVibe)
  ├─ PlayerView → Tone.js engine → 无限循环音乐
  ├─ VibeBackground → 内置 Canvas 兜底视觉
  └─ VibeCanvasFrame → iframe sandbox 渲染 AI visualCode
```

AI 输出的 `visualCode` 不直接进入主页面执行，而是进入 `<iframe sandbox="allow-scripts">`。组件会在注入前阻断明显危险 API（外部请求、`document.cookie`、`localStorage`、`top`/`parent` 访问等）；运行失败时自动不显示 AI 视觉，保留内置背景。

Tone.js 引擎通过 `Tone.Analyser` 输出 FFT。`Page` 用 `requestAnimationFrame` 读取 FFT，并同时传给 `VibeBackground` 与 `VibeCanvasFrame`，从而让内置视觉和 AI 视觉都能跟随真实音乐律动。

## 6. 组件详解

### 6.1 `page.tsx` — 页面状态机

承担「首页 ↔ 播放器」的切换。通过 CSS 自定义属性把当前 vibe 的调色板注入到 `.stage`（`--accent` / `--accent-2` / `--base`），使整页的氛围灯光、滑块、按钮颜色随 vibe 联动。

### 6.2 `VinylSceneSelector.tsx` — 3D 唱片选择器

首页主交互，基于 Three.js 的 `WebGLRenderer`。

**渲染内容**：

- 每个 vibe 生成一组 `THREE.Group`，包含封套（`BoxGeometry` + 动态生成的封面贴图）、黑胶唱片（`CylinderGeometry`）、标签、5 圈纹路（`TorusGeometry`）。
- 封面贴图由 `createCoverTexture()` 用离屏 Canvas 即时绘制：渐变背景 + 同心圆 + 氛围名 + 副标题 + 巨型 glyph，再转为 `CanvasTexture`。

**交互**：

- 指针移动 → 整组轻微视差旋转（`pointerRef`）。
- 拖拽（`dragRef`）超过阈值 → 切换上一/下一张。
- 箭头按钮、键盘方向键、底部 chip 队列均可切换；Enter 播放。

**动画**：`requestAnimationFrame` 中用 `shortestDelta` 计算每张唱片相对当前索引的环形位移，做位置/缩放/旋转的阻尼插值（lerp 系数 0.085），中心唱片放大并加快自转。

**性能与清理**：

- `setPixelRatio(min(dpr, 1.75))` 限制像素比。
- 仅渲染中心 ±2 张（`item.group.visible = distance <= 2`）。
- 尊重 `prefers-reduced-motion`。
- `useEffect` 清理函数中 `dispose()` 所有几何体、材质、贴图与 renderer，避免 WebGL 内存泄漏。

### 6.3 `PlayerView.tsx` — 播放器视图

进入某个 vibe 后的主界面，分三块：

- **顶部**：返回按钮、英文副名、`查看代码` 开关。代码预览（`codepeek`）把当前参数渲染成一段**只读**的伪 DSL（`vibe("id").tempo(..).energy(..)...`），呼应规划中的「代码即音乐」/ live coding 方向。
- **中部**：glyph、氛围名、tagline、播放状态（当前文案为「正在生成无限循环 · 静音预览」，明确标示尚未出声）。
- **底部 dock**：
  - Transport 面板：播放/暂停按钮 + Tempo 显示。
  - 参数面板：遍历 `PARAM_META` 渲染 7 个 `Slider`。

### 6.4 `VibeBackground.tsx` — 实时视觉背景（核心）

全屏 Canvas 2D 动画，是产品「视觉主角」。详见下一节。

### 6.5 `Slider.tsx` — 受控滑块

无状态受控组件。`unit` 存在时按整数显示（BPM），否则显示 0~100 的百分比；通过 `--fill` CSS 变量绘制填充进度。

### 6.6 备选组件

- `RecordPlayerSelector.tsx`：纯 CSS 黑胶转盘 + 唱臂 + 横向唱片架，无 WebGL 依赖，可作为低端设备降级方案。
- `VibeCard.tsx`：经典卡片式氛围入口。

二者当前未挂载，保留作为不同交互形态的备选实现。

## 7. 实时视觉渲染原理

`VibeBackground` 是理解项目「参数如何变成画面」的关键，也是将来接入音频的衔接点。

### 7.1 设计要点

- 用 `useRef` 持有最新 props（`state.current`），动画循环 `useEffect` 依赖数组为空 `[]`，**只建立一次** `requestAnimationFrame` 循环，避免每帧重建。
- 颜色取自 `vibe.palette`，经 `hexToRgb` 转换后用于 `rgba` 叠加。
- 粒子数量随 `params.density` 缩放：`60 + density * 140`。
- 尊重 `prefers-reduced-motion`：减弱或停止运动。

### 7.2 伪律动（音频接入点）

当前没有声音，于是用时间驱动的正弦函数模拟「呼吸 / 律动」：

```ts
// 伪律动：将来替换成真实 FFT 振幅
const breath = playing ? 0.5 + 0.5 * Math.sin(t * 2.2) : 0.18;
```

> 🔌 **接入 Tone.js 时**：把 `breath`（以及各图层用到的振幅量）替换为 `Tone.Analyser` / FFT 的真实频谱数据即可，视觉会自动跟随真实音乐起伏，无需重写渲染管线。

### 7.3 渲染分层

1. **基底**：纯色背景填充。
2. **氛围光球**：3 个跟随时间缓慢移动的径向渐变（`globalCompositeOperation = "lighter"`），半径受 `space`、亮度受 `brightness` 与 `breath` 调制。
3. **标志性图层**（按 `vibe.visual` 分支）：
   - `rain`：斜向雨丝线条，强度受 `ambience`。
   - `waves`：5 条相位错开的正弦曲线，振幅受 `space` + `breath`。
   - `cosmos`：闪烁星点（`lighter` 叠加）。
   - `orbs` / `particles`：漂浮粒子，速度受 `energy`；`orbs` 额外绘制近邻连线（距离 < 120px）。

参数到视觉的映射关系总览：

| 参数 | 主要影响 |
| --- | --- |
| `energy` | 粒子运动速度、整体动效速度 |
| `space` | 光球半径、波浪振幅 |
| `brightness` | 光球/粒子透明度、星点亮度 |
| `density` | 粒子数量 |
| `ambience` | 雨丝强度 |
| `tempo` | 当前仅在 UI 显示（未驱动视觉节拍，待音频接入后统一） |
| `warmth` | 当前主要体现在调色板选择上（未单独驱动视觉） |

## 8. 设计系统与样式约定

样式集中在 `src/app/globals.css`（约 1500 行，原生 CSS，无 Tailwind/CSS-in-JS）。

- **CSS 变量驱动主题**：`:root` 定义基础令牌（`--base` / `--accent` / `--fg` / `--glass` / `--ease` 等）；`page.tsx` 在运行时把当前 vibe 的调色板写入 `.stage` 的内联变量，实现整页随 vibe 换色。
- **字体**：`--font-sans`（Space Grotesk）、`--font-mono`（JetBrains Mono），由 `layout.tsx` 注入；`.mono` 类切换等宽字体（用于代码/数值）。
- **氛围灯光层**：`.stage__vignette`（暗角）、`.stage__grain`（颗粒）叠加在 Canvas 之上营造质感。
- **玻璃拟态**：面板使用 `--glass` / `--glass-border` 半透明 + 模糊风格。
- **缓动**：统一使用 `--ease: cubic-bezier(0.22, 1, 0.36, 1)`。

> 命名采用 BEM 风格（`.player__top`、`.vinyl-selector__head`、`.card__glyph`）。

## 10. 技术演进路线图

对齐 [`docs/想法.md`](./想法.md) 的开发节奏，按优先级排列。

### 阶段一：让它出声（最高优先级）

引入 Tone.js，把现有 `VibeParams` 真正接到声音上。建议新增模块（对应规划的 `packages/audio-engine`）：

```txt
src/audio/
  engine.ts      # Tone.Transport / 合成器 / 效果器 的封装
  layers.ts      # drums / bass / pad / melody / ambience 各层生成
  mapping.ts     # VibeParams → 合成参数（tempo→bpm, space→reverb.wet, ...）
  analyser.ts    # Tone.Analyser，输出 FFT 供 VibeBackground 使用
```

接入要点：

1. 用户首次播放需 `await Tone.start()`（浏览器音频上下文需用户手势激活）。
2. `changeParam` 时同步更新合成器参数，实现实时调音。
3. 把 `analyser` 的频谱接到 `VibeBackground` 的 `breath`，视觉与声音同步。

### 阶段二：产品化

- **分享链接**：把 `VibeParams`（+ vibe id）编码进 URL query/hash，配合 `getVibe()` 还原。
- **Preset 导出/导入**：序列化为 JSON 下载/粘贴。
- **状态管理迁移**：引入 Zustand/Jotai，统一参数源。

### 阶段三：社区

- 用户生成 preset、收藏、复制代码、排行榜。

### 阶段四：Live coding

- 把 `PlayerView` 中只读的「代码预览」升级为可编辑编辑器，自研简化 Pattern DSL，或嵌入 Strudel（注意 AGPL/GPL 许可证义务）。

### 风险与注意事项（来自规划）

- 不要第一版就做完整 DAW（钢琴卷帘、多轨、自动化会拖死项目）。
- 不要一开始就做大模型音乐生成（版权、算力、延迟、质量难控）。先做算法音乐，后期再接 Magenta.js 或服务端模型。
- **许可证**：Tone.js（MIT）最适合产品化；Strudel / Hydra / Tidal 生态含 AGPL/GPL，深度集成进线上服务时需保持开源并保留许可证声明。
